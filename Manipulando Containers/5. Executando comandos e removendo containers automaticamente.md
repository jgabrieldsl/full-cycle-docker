# Executando Comandos e Remoção Automática

Nesta etapa, exploramos como interagir com containers que já estão em execução e como manter nosso ambiente limpo removendo containers automaticamente.

## Docker Exec: Comandos em Containers Ativos

Frequentemente confundido com o `docker run`, o comando `exec` serve para rodar processos dentro de um container que **já está rodando**.

- **docker run:** Cria e inicia um **novo** container.
- **docker exec:** Executa um comando em um container **existente**.

### Exemplos Básicos
```bash
# Listar arquivos dentro do container
docker exec <ID_OU_NOME> ls

# Listagem detalhada
docker exec <ID_OU_NOME> ls -la
```

## Interatividade e Terminal (-it)

Para "entrar" no container e interagir com seu terminal, utilizamos os parâmetros `-i` e `-t` combinados com um shell (como `bash` ou `sh`):

```bash
docker exec -it <ID_OU_NOME> bash
```

- **-i (interactive):** Mantém o STDIN aberto mesmo se não estiver atachado.
- **-t (tty):** Aloca um pseudo-terminal (conecta seu teclado ao terminal do container).

### Uso em Desenvolvimento
O `docker exec -it` é essencial no dia a dia do desenvolvedor. Ele permite utilizar ferramentas instaladas apenas no container (como Python, Node, MySQL, etc.) sem precisar instalá-las na sua máquina local.

> [!WARNING]
> **Cuidado em Produção:** Evite alterar arquivos ou configurações diretamente dentro de um container em produção. Containers são efêmeros; qualquer mudança manual será perdida se o container for reiniciado ou substituído.

## Remoção Automática com --rm

Por padrão, quando um container para (via `Ctrl+C` ou fim do processo), ele continua ocupando espaço no sistema no estado `Exited`. Para evitar o acúmulo de "lixo", usamos a flag `--rm`:

```bash
docker run -it --rm nginx
```

- **--rm:** Garante que o container seja **removido automaticamente** assim que o processo principal terminar.

### Limpeza Manual de Múltiplos Containers
Se você já acumulou muitos containers, pode removê-los passando vários IDs ao comando `rm`:

```bash
docker rm -f <ID1> <ID2> <ID3>
```
