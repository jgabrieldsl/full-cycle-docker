# Nomeando containers e entendendo diferentes execuções

Depois da execução do nosso primeiro container, é importante destacar alguns pontos fundamentais sobre como o Docker gerencia esses containers.

## Listando Containers

Toda vez que executamos o comando:

```bash
docker ps -a
```

O Docker lista todos os containers (ativos ou não). Alguns pontos importantes:
- **Nomes Automáticos**: Se você não especifica um nome, o Docker gera um automaticamente (como `clever_hawking`) para evitar conflitos.
- **ID do Container**: Além do nome, cada container possui um ID único.
- **Nomes são Obrigatórios**: Todo container precisa ter um nome por padrão.

## Atribuindo Nomes aos Containers

Para definir um nome específico ao criar um container, utilizamos o parâmetro `--name`:

```bash
docker run --name meu-container hello-world
```

### O Comportamento do `docker run`

Uma confusão comum é achar que o `docker run` reexecuta um container anterior se usarmos a mesma imagem. **Isso não é verdade.** 

O comando `docker run` sempre **cria um novo container**, independentemente de já existir outro com a mesma imagem. Eles são instâncias completamente independentes.

## Explorando a Documentação e Filtros

Para ser independente no Docker, é essencial saber usar o `--help`. O parâmetro `--name` não pertence ao comando principal `docker`, mas sim ao subcomando `run`.

Para ver todas as opções do `run`:
```bash
docker run --help
```

Como a lista de parâmetros é imensa, uma dica de ouro é filtrar a saída usando o `grep`:
```bash
docker run --help | grep name
```
Isso retornará apenas as linhas que contêm a palavra "name", facilitando a localização do parâmetro desejado.

## Dica de Organização: Usando o Sinal de Igual

Quando lidamos com muitos parâmetros, a visualização pode ficar confusa. Para melhorar a legibilidade, você pode usar o sinal de `=` entre o parâmetro e seu valor:

```bash
docker run --name=meu-container-2 --param1=valor1 --param2=valor2 hello-world
```
Isso ajuda a distinguir claramente o que é o nome do parâmetro e o que é o valor atribuído a ele.

## Executando Comandos dentro do Container

A estrutura padrão do comando é:
`docker run [parâmetros] [imagem] [comando]`

### Comando Padrão vs. Comando Forçado

Ao rodar `docker run hello-world`, ele executa um comando padrão definido pelo criador da imagem (neste caso, o executável `/hello`).

Podemos "forçar" a execução de outro comando:
```bash
docker run --name=meu-container-3 hello-world /hello
```

### Falhas na Execução

Se tentarmos executar um comando que não existe dentro da imagem (ex: `XPTO`):
```bash
docker run --name=meu-container-erro hello-world XPTO
```
O Docker criará o container, mas ele falhará ao iniciar porque não encontrou o executável especificado. Ao rodar `docker ps -a`, você verá le container na lista, mas com um status de erro, pois o processo principal nunca chegou a rodar com sucesso.
