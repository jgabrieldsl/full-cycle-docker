# Executando nosso primeiro container com Hello World

Chegou a hora de rodar os nossos principais comandos e entendermos o comportamento que eles estão realizando. O passo inicial quando trabalhamos com Docker é executar o primeiro container.

## O Comando `docker run`

Para começar, utilizamos o comando básico:

```bash
docker run hello-world
```

Embora pareça simples, esse comando realiza diversas operações "por baixo dos panos". Quando você o executa, o Docker exibe logs, incluindo a mensagem "Hello from Docker!", e finaliza.

Ao usar o `docker run` seguido do nome de uma imagem, você está especificando qual "molde" será utilizado para criar o container.

## Como o Docker gerencia as Imagens?

Toda vez que você executa `docker run`, o Docker segue este fluxo:

1.  **Verificação Local:** Verifica se a imagem já existe na sua máquina. Se não for encontrada, ele avisa (ex: `Unable to find image... locally`).
2.  **Tags (Versões):** Note que ele busca por `hello-world:latest`. Quando não especificamos uma versão, o Docker assume automaticamente a tag `:latest` (a mais recente). O formato é sempre `nome-da-imagem:versão`.
3.  **Download (Pull):** Se não estiver localmente, o Docker baixa a imagem de um **Registry**.

### O que é um Registry?
O registry padrão é o **Docker Hub** (docker.io). No entanto, você pode usar outros serviços como:
*   Registries privados (da sua empresa).
*   **AWS** Container Registry.
*   **Azure** Container Registry.
*   **Google Cloud** Registry.

## Camadas e o Overlay File System

Uma imagem Docker não é um arquivo único e massivo, mas sim um conjunto de arquivos organizados em **camadas (layers)**.

*   **Overlay File System:** O Docker baixa a imagem em "pedacinhos".
*   **Reutilização:** Cada camada possui um hash único. Se você baixar uma imagem do Ubuntu e depois uma do Alpine que compartilhe camadas em comum, o Docker baixará apenas as diferenças, economizando espaço e tempo.
*   **Extração:** Após baixar todas as camadas (ou layers), o Docker as combina e extrai para rodar o container.

## Ciclo de Vida e Processos

Um ponto fundamental: **O container só permanece rodando enquanto o seu processo principal estiver ativo.**

1.  **Exemplo Hello World:** O container executa um binário que apenas imprime uma mensagem e finaliza. Como o programa acabou, o container para automaticamente.
2.  **Exemplo Servidor Web (Nginx):** Um servidor web precisa ficar "ouvindo" requisições. Enquanto o processo do Nginx estiver ativo, o container continuará rodando.
3.  **Entrypoint:** É o comando configurado dentro da imagem para ser executado assim que o container inicia.

## Gerenciando Containers

### Listando containers ativos
Para ver os containers que estão em execução no momento:
```bash
docker ps
```

### Listando todos os containers (incluindo parados)
Mesmo que um container tenha finalizado (como o `hello-world`), ele ainda existe no seu computador. Para visualizá-lo:
```bash
docker ps -a
```

Neste comando, você verá informações importantes:
*   **Container ID:** Um identificador único (fingerprint) em hexadecimal.
*   **Image:** A imagem utilizada (`hello-world`).
*   **Command:** O comando executado no entrypoint (ex: `/hello`).
*   **Status:** Há quanto tempo foi criado e quando saiu (Ex: `Exited (0) 9 minutes ago`).
*   **Names:** Um nome aleatório atribuído pelo Docker (se você não definir um).
