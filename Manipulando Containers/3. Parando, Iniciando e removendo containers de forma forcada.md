# Ciclo de Vida: Parar, Iniciar e Remover Containers

Nesta etapa, aprofundamos no gerenciamento do ciclo de vida dos containers, entendendo como parar processos, reiniciar containers existentes e realizar remoções seguras ou forçadas.

## Removendo Containers Parados

Quando um container finaliza sua tarefa ou está parado, podemos removê-lo do sistema para liberar espaço.

### 1. Remoção por ID ou Nome
Você pode usar o comando `docker rm` seguido do ID (ou prefixo) ou do nome personalizado:

```bash
# Por ID
docker rm <CONTAINER_ID>

# Por Nome
docker rm meu-container
```

> [!IMPORTANT]
> Por padrão, o Docker só permite remover containers que estão no estado **parado** (`Exited`).

## Containers em Execução (Exemplo com Nginx)

Diferente de imagens como o `hello-world` que executam e param, o **Nginx** é um servidor web que mantém o container rodando indefinidamente.

```bash
docker run --name meu-nginx nginx
```

Ao verificar o status em outro terminal com `docker ps`, você verá o status como `Up` (em execução).

## Parando e Reiniciando Containers

### Parar (Stop)
Se tentarmos remover um container ativo, o Docker retornará um erro. O procedimento correto é interrompê-lo primeiro:

```bash
docker stop meu-nginx
```
O `docker stop` envia um sinal ao processo interno para que ele se encerre de forma "saudável" (graceful shutdown).

### Iniciar (Start)
Diferente do `run`, o `start` não cria nada novo, apenas religa o que já existe:

```bash
# Inicia um container que já foi criado anteriormente
docker start meu-nginx
```

| Comando | Função |
| :--- | :--- |
| **`docker run`** | Cria e inicia um **novo** container. |
| **`docker start`** | Inicia um container **existente** que estava parado. |

## Remoção Forçada (Force Remove)

Para remover um container que está em execução sem precisar pará-lo primeiro, utilizamos a flag `-f`:

```bash
docker rm -f meu-nginx
```
O **Force Remove** interrompe o processo abruptamente e remove o container do sistema imediatamente.

---

### Resumo Rápido
- **`docker ps`**: Lista containers ativos.
- **`docker ps -a`**: Lista todos os containers (ativos e parados).
- **`docker stop`**: Para o container de forma segura.
- **`docker rm`**: Remove containers parados.
- **`docker rm -f`**: Força a remoção de containers (mesmo ativos).
