# Removendo Todos os Containers com Subcomandos

À medida que trabalhamos com Docker, é comum acumularmos dezenas de containers parados ou em execução. Removê-los um a um manualmente pelo ID é um processo lento e ineficiente.

## O Desafio da Remoção em Lote

Imagine que você executou o `hello-world` diversas vezes ou subiu vários ambientes de teste. Ao rodar `docker ps -a`, você verá uma lista extensa. Em vez de copiar cada ID e rodar `docker rm`, podemos automatizar esse processo utilizando **subcomandos**.

## Listando apenas IDs com a flag -q

O primeiro passo para a automação é fazer o Docker retornar apenas o que precisamos: os identificadores dos containers. Para isso, usamos a flag `-q` (**quiet**):

```bash
# Lista apenas os IDs de todos os containers (ativos e inativos)
docker ps -aq
```

- **-a:** Lista todos os containers.
- **-q:** Retorna apenas os IDs (hashes).

## Combinando Comandos (Subcomandos)

Podemos passar o resultado de um comando como argumento para outro. No terminal, fazemos isso envolvendo o subcomando em `$( )`:

```bash
# Remove forçadamente todos os containers do sistema
docker rm -f $(docker ps -aq)
```

### Como funciona:
1. O shell executa primeiro o que está dentro de `$(docker ps -aq)`.
2. O resultado (uma lista de IDs) é "colado" logo após o `docker rm -f`.
3. O comando final de remoção é executado com todos os IDs de uma vez.

> [!TIP]
> Essa técnica de composição de comandos não serve apenas para o Docker; é um conceito fundamental do Shell Script que pode ser usado para quase qualquer tarefa de automação no Linux/Unix.

## Resumo da "Malícia"

- Para remover **todos** (ativos e inativos): `docker rm -f $(docker ps -aq)`
- Para remover apenas os que estão **parados**: `docker rm $(docker ps -q)` (sem o `-a`)

Com essa dica, você economiza tempo e mantém seu ambiente de desenvolvimento sempre limpo.
