# Attach e Detach

Ao executar containers, é fundamental entender como o Docker lida com a conexão entre o terminal e o processo em execução.

## Execução em Primeiro Plano (Foreground)

Por padrão, ao executar um comando como:

```bash
docker run nginx
```

O processo é mantido em execução e o terminal fica **bloqueado**, exibindo os logs em tempo real. Se você pressionar `Ctrl + C`, o container recebe um sinal de interrupção, para sua execução e sai (`Exited`).

Embora seja útil para visualizar logs imediatamente, manter o terminal preso é inconveniente para gerenciar múltiplos containers.

## Execução em Segundo Plano (Detached Mode)

Para liberar o terminal e deixar o container rodando em background, utilizamos a flag `-d` (**detached**):

```bash
docker run -d nginx
```

- **Detached (-d):** "Desatacha" o terminal do processo.
- O Docker retorna apenas o **ID (hash)** do container criado.
- O container continua rodando de forma independente.

### Gerenciando Containers em Background

Você não precisa usar o ID completo para referenciar um container. O Docker permite usar apenas os primeiros caracteres, desde que sejam únicos:

```bash
# Removendo um container pelo início do ID
docker rm -f 10f
```

## Reatachando ao Terminal (Attach)

Se um container foi iniciado em modo detached, você pode voltar a visualizar sua saída padrão (stdout) usando o comando `attach`:

```bash
# Comando correto para reatachar
docker attach <ID_OU_NOME>
```

> [!IMPORTANT]
> Cuidado: `docker run attach` tentará criar um novo container chamado "attach". O comando correto para se conectar a um container existente é apenas `docker attach`.

### Comportamento do Attach
- Ao usar `docker attach`, seu terminal volta a exibir os logs do container.
- Se você pressionar `Ctrl + C` enquanto estiver "atachado", o sinal será enviado ao processo principal do container, **encerrando-o**.

## Resumo

- **Modo Padrão (Attach):** O terminal fica preso ao container. `Ctrl + C` encerra o container.
- **Modo Detached (`-d`):** O container roda em segundo plano e libera o terminal.
- **Comando `attach`:** Permite alternar um container de background para foreground.
