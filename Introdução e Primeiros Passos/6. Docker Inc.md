## Docker Inc

O Docker é uma ferramenta open source, mas a Docker Inc tem um foco específico para o desenvolvedor. Então, esse é o ponto importante. Toda vez que você lembrar da Docker Inc, ela é uma empresa que tem o objetivo de ajudar o desenvolvedor a ser mais produtivo no ecossistema de containers. Eles chamam isso de inner circle. Basicamente, existem o inner circle e o outer circle. O outer circle representa o momento em que algo é colocado em produção, ou seja, quando sai do ciclo de desenvolvimento do software. E o inner representa o que acontece no desenvolvimento do software. Então, ele quer ajudar o desenvolvedor nesse aspecto aqui.

E por conta disso, ele tem um foco muito grande em aprimorar o Docker Engine ou Community Edition, melhorar o Docker, então ela investe muito dinheiro na ferramenta Docker, que já é open source, mas também em ferramentas como o Docker Desktop, que facilitam a vida do desenvolvedor e também onde ela ganha dinheiro. Então, a Docker Inc, a cada dia, constrói um ecossistema em torno do Docker também, e no mundo dos containers, para ajudar os desenvolvedores. E, para isso, eles começaram a comprar empresas e projetos. Quando a gente fala na Docker Inc, ela não está interessada apenas em rodar o seu container. Ela está interessada em fazer com que você gerencie isso tudo de uma forma mais fácil, mas também tudo que está em volta de um container. Por exemplo, a Docker Inc tem o Testcontainers. Ela comprou uma empresa que fez o Testcontainers para que a gente consiga desenvolver no dia a dia os nossos testes, mas que esses testes consigam ser integrados end-to-end, onde, quando eu rodo um teste, ele sobe os containers com aplicações de dependência, como um RabbitMQ, um banco de dados. Assim que você rodou o seu teste esses containers caem. Assim, que fica em volta de um container, a Docker está interessada. Novamente, ela quer ajudar o desenvolvedor. E, para isso, eles têm alguns produtos. 

### Docker Desktop
Docker Desktop (inclusive, a gente vai ver isso nesse curso. A gente vai te ensinar a instalar essa ferramenta, que vai fazer com que você rode o Docker no dia a dia no seu computador de uma forma bem descomplicada, bem tranquila). E essa ferramenta, o Docker Desktop, tem uma versão gratuita, pessoal, ou seja, se você é desenvolvedor, quer trabalhar na sua máquina normal, você pode utilizar o Docker Desktop sem custo algum. Mas, lembrando, não é open source. Tem planos como um plano PRO que você mesmo pode comprar ou coisas desse tipo que fazem com que você tenha ainda mais recursos para trabalhar com o Docker Desktop. Desde recursos para velocidade, sincronização de arquivos até armazenamento de imagens privadas em container registries como o Docker Hub.

### Docker Hub
Depois disso, a gente tem o Docker Hub, que é uma outra ferramenta, um ecossistema da Docker, onde você pode armazenar essas imagens e, inclusive, acessar imagens mais populares do mercado para você sair utilizando no dia a dia. Então, com o Docker Hub, você vai ver que existem imagens verificadas, imagens oficiais, imagens de empresas que trabalham, que mantêm imagens, vamos dizer assim, não oficiais da Docker, mas são empresas confiáveis e tudo mais. E você vai perceber que o Docker Hub é um mundo de possibilidades que você tem para navegar e entender tudo que tem disponível no mundo dos containers.

### Docker Build Cloud
O Docker também tem o Build Cloud, que faz o empacotamento dessas imagens de uma forma muito mais rápida do que qualquer esteira de CI/CD ou qualquer coisa desse tipo. Normalmente, quando você vai colocar uma aplicação em produção você precisa pegar essa imagem, pegar a sua aplicação e empacotar. Esse empacotamento, que a gente chama de build, muitas vezes é um pouco custoso, pode ser demorado, e muitas vezes a gente precisa fazer ele em diversas arquiteturas, como ARM, AMD64. Então, o Build Cloud é uma ferramenta que você pode integrar na sua esteira de CI/CD ou mesmo rodar no local na sua máquina, mas ele faz esse empacotamento, então o processo é muito mais rápido. Isso é mais uma ferramenta que ele tem, e também tem versão paga.

### Docker Scout
Um serviço  que eu tiro o chapéu é chamado de Docker Scout, que é focado na área de segurança. Então, toda vez que você cria uma imagem, uma aplicação, ele consegue varrer aquilo e trazer, de forma extremamente detalhada, tudo que tem de vulnerabilidade de segurança. Você consegue ver se aquela vulnerabilidade já foi corrigida e, se foi, você pode corrigir naquela imagem. Tem recursos fantásticos e que eu acho um grande diferencial competitivo da Docker Inc.

### Docker AI
Tem a parte de Docker AI, que é uma iniciativa deles para conseguir rodar softwares e ferramentas de AI de forma containerizada.

### Docker Acquisitions
Também, ao longo do tempo, você vai perceber que a Docker adquiriu algumas empresas e projetos. Por exemplo, ela adquiriu uma empresa chamada InfoSifter. Essa empresa tinha um foco muito grande em segurança e verificação de imagem. Então, foi comprada e grande parte do know-how dela foi para o Docker Hub, onde você tem, por exemplo, Verified Publishers, que são fornecedores que você consegue confiar mais nessas imagens. Essa empresa acaba ajudando em relação a todo esse processo. Também comprou a Tilt. A Tilt é uma empresa focada para você fazer desenvolvimento de aplicações, normalmente baseadas em microsserviços. Quando você tem aplicações que rodam com dependências com muitas outras aplicações, você precisa de tudo isso no ar para conseguir trabalhar.

### Kubernetes
Outra coisa é que existe uma ferramenta chamada Kubernetes, que é um orquestrador de containers. Vamos imaginar que ele é um Docker muito focado para execução de containers e escala de containers de forma distribuída, com várias máquinas e tudo mais. Hoje em dia, quando a gente fala em containers, principalmente em produção, falamos sobre Kubernetes. Essa empresa ajuda você a desenvolver com o Kubernetes local na sua máquina e, cada vez que você salva, cada vez que você modifica, ele já faz deploy no Kubernetes para você ver esse comportamento em tempo real, inclusive com as dependências que essas aplicações acabam tendo.

Ela comprou também a AtomicJar, responsável pela criação do Testcontainers, que falei um pouco para vocês, permitindo desenvolver aplicações e, na hora de criar seus testes, utilizar containers. Isso muda o jogo para desenvolvimento. Essa AtomicJar tem o Testcontainers e iniciativas como o Testcontainers Cloud. Então, quando eu rodo meus testes, em vez de rodar esses testes na minha máquina e subir os containers nela, às vezes não tenho um recurso computacional tão forte, ele consegue rodar isso na nuvem de forma muito rápida, com baixa latência. Tem também a ferramenta Testcontainers Desktop. Então, é uma aquisição bem interessante e dá para perceber que o Docker não está focado apenas em executar containers. Ele está focado, por exemplo: em você publicar suas imagens com segurança, desenvolver de forma mais integrada com Kubernetes e desenvolver seus testes de forma mais simples e integrada.

### Mutagen.io
Por fim, a Docker comprou uma empresa chamada Mutagen.io, que criou um recurso extremamente fantástico chamado Synchronized File Shares. Imagine que você tem containers com mais de 100 mil arquivos. Quando está rodando um container no seu computador, principalmente se você não está trabalhando com Linux, precisa de uma máquina virtual Linux para rodar os containers. Mas, às vezes, é necessário compartilhar arquivos da sua máquina com o container. Para isso, esses arquivos precisam passar pela máquina virtual, o que pode gerar latência, especialmente com muitos arquivos. Hoje, essa latência é muito pequena, mas com essa tecnologia avançada de cache para sincronização de arquivos, o processo é praticamente imperceptível.
