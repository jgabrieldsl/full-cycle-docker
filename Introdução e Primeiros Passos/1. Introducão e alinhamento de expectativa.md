Meu nome é Wesley. Seja muito bem-vindo ao nosso curso de Docker na Prática. Este curso tem o objetivo de fazer com que você tenha um alto nível de consciência e entendimento sobre containers, tanto na prática quanto na teoria. O ponto principal é que, com este curso, não tenho dúvidas de que você sairá com um nível de profundidade elevado para trabalhar com Docker em qualquer lugar. Este vídeo, além de ser introdutório, serve para alinharmos expectativas e para que você saiba o que será capaz de fazer ao final do curso.

Um dos pontos importantes que quero destacar é que, ao falarmos sobre containers, precisamos entender a diferença entre o Docker, como ferramenta e runtime de containers, e o conceito de containers propriamente dito. A ideia aqui é que você compreenda os principais pontos para que, sempre que tratarmos de containers, você saiba o que está acontecendo por trás desse ecossistema.

Por isso, não podemos deixar de abordar containers, Docker, máquinas virtuais e a confusão comum no mercado entre o Docker como ferramenta e a Docker Inc., a empresa. Assim, exploraremos a história do Docker e aprenderemos a instalar a ferramenta em múltiplos sistemas operacionais. A partir daí, colocaremos a mão na massa para manipular containers, o que inclui criar, executar, remover e publicar portas, permitindo que você acesse os containers da sua máquina.

Vamos aprender a executar servidores e acessar containers em execução para trabalhar diretamente dentro deles. Como nosso foco principal é o desenvolvimento, falaremos bastante sobre volumes e bind mounts, que permitem compartilhar o conteúdo do seu projeto com o container. Para isso, abordaremos também imagens Docker e registries, os repositórios onde as imagens ficam armazenadas, além de ensinar como criar suas próprias imagens Docker, fundamentais para rodar containers.

Você aprenderá a empacotar sua aplicação em imagens, que são imutáveis. Existem formas diferentes de realizar esse empacotamento, conhecidas como builds. O Docker oferece ferramentas como o buildx, que possui recursos avançados pouco utilizados por muitos desenvolvedores. Esses recursos incluem builders personalizados para facilitar o processo de criação de imagens.

Outro ponto essencial é a comunicação entre containers. Vamos aprender como configurar redes, permitindo que diferentes aplicações e bancos de dados interajam entre si. Isso inclui conhecer tipos de redes e suas configurações. Depois de entender essa base, avançaremos para um nível mais alto, onde, com poucas linhas de um arquivo declarativo, será possível gerenciar múltiplos containers simultaneamente, facilitando sua vida e evitando que você fique digitando por muito tempo para trabalhar com containers. Para isso, utilizaremos o Docker Compose, uma ferramenta fantástica para lidar com aplicações multi-container.

Também fizemos questão de colocar no curso o que você precisa para conseguir entender a nossa ideia sobre o Docker Desktop, uma ferramenta da Docker Inc., que facilita o gerenciamento de containers e inclui recursos para análise de segurança de imagens antes da publicação.

E, por último, vamos falar sobre uma ferramenta que  poucas pessoas conhecem, mas que é algo que pode facilitar quando você começar um novo projeto trabalhando com containers ou, mesmo, se você já tem um projeto criado, você conseguir containerizar esse seu projeto com um único comando, que é o que a gente chama de Docker Init.

Então, percebemos que tem bastante conteúdo. Percebemos que a gente vai se aprofundar em pontos que são necessariamente mais focados no mundo do desenvolvimento com Docker. Então, a nossa ideia aqui não é que você coloque aplicações Docker em produção. Obviamente, vamos ter todas as boas práticas para você fazer isso, mas a nossa ideia é que você tenha um alto nível de consciência da ferramenta, um aprofundamento muito bom para que, em qualquer lugar que você vá, você consiga trabalhar com Docker. Assim, esse é o nosso objetivo, e eu não tenho dúvidas de que você vai gostar bastante do que a gente acabou de fazer.
