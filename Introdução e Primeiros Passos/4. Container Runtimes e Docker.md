## Resumo da Aula

Agora que entendemos mais sobre como funciona a ideia de container e a diferença entre uma máquina virtual e um container, vamos falar sobre o Docker e os container runtimes. Primeiramente, container runtime é um software que permite a execução de containers. Isso significa que ele possui todas as especificações necessárias para que um container possa ser executado. Atualmente, essas especificações são padronizadas, e ao longo do curso veremos como isso funciona.

A Docker, junto com outras empresas, criou há alguns anos a iniciativa chamada Open Containers Initiative. Essa iniciativa estabeleceu uma padronização que permite que qualquer container seja executado da mesma forma, independentemente do container runtime utilizado. Por exemplo, se você usar um container runtime diferente do Docker, como o Podman, poderá executar o mesmo container sem problemas. Essa padronização é fundamental para garantir a interoperabilidade entre os diferentes runtimes e facilitar o uso de containers em diversos ambientes.

Toda vez que falarmos em container runtime, estamos nos referindo à ferramenta que executa containers e que segue especificações para garantir a compatibilidade. É importante entender esse conceito, e, como o curso é focado no Docker, trarei mais informações sobre essa tecnologia específica, sem entrar em detalhes sobre outros runtimes. No entanto, é relevante destacar que o Docker não foi o primeiro a rodar aplicações em containers. Em 2008, por exemplo, surgiu o LXC (Linux Containers), que utilizava recursos do Linux, como namespaces,  e cgroups. Os namespaces permitem o isolamento de processos, rede e outros elementos, enquanto os cgroups possibilitam o controle dos recursos computacionais alocados, como limite de CPU ou memória. Com essas funcionalidades, era possível isolar processos e definir limites para seu uso de recursos, como apenas uma vCPU ou 500 MB de memória.

O Docker, criado posteriormente por uma empresa que será abordada mais adiante no curso, revolucionou a forma de trabalhar com containers. Não é possível falar sobre Docker sem mencionar a história dessa empresa, pois elas estão profundamente interligadas. Apesar disso, é fundamental entender que o conceito de container não foi criado pelo Docker, mas sim aprimorado por ele.

A Open Containers Initiative pode ser acessada em opencontainers.org, onde você encontrará mais informações sobre essa padronização. Embora o Docker e a empresa Docker sejam altamente relevantes no mundo dos containers, é importante lembrar que eles não foram os criadores do conceito.
